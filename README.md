# МагӀарул мацӀ — Аварско-русский и Русско-аварский словарь

## О проекте
Полнофункциональный онлайн-словарь аварского языка с интеграцией в Telegram и веб-приложением (Mini App). Проект обеспечивает быстрый и надежный поиск переводов с учетом морфологии, предоставляет ежедневную рассылку "Слово дня" и позволяет использовать словарь как обычное веб-приложение.

**Репозиторий проекта:** [https://github.com/sharapik26/Avar_dictionary](https://github.com/sharapik26/Avar_dictionary)

## Состав команды
- **Насрулаев Шарапудин**
- **Мусаев Умахан**
- **Магомедов Абдул**
- **Атаханов Набиюлла**

## Основной функционал
1. **Многоуровневый поиск**: Поддержка двустороннего поиска (Аварско-русский, Русско-аварский).
2. **Telegram Bot**: Бот, который переводит слова прямо в чате.
3. **Telegram Mini App**: Красивое, адаптивное веб-приложение, открывающееся прямо внутри Telegram (или в браузере).
4. **Слово дня**: Ежедневная автоматическая рассылка случайного слова с примером для всех подписавшихся пользователей (работает через APScheduler).
5. **API-сервер**: Быстрый бэкенд на базе FastAPI с отдачей JSON-ответов и статики.

## Робастность и отказоустойчивость
- В коде API и Telegram-бота предусмотрена обработка пустых или невалидных запросов.
- Реализована надежная архитектура: единый монолит на FastAPI (`Lifespan`), который запускает и API-сервер, и Telegram-бота в одном процессе.
- При отправке рассылки "Слово дня" бот перехватывает ошибки (например, если пользователь заблокировал бота), удаляя невалидные ID из базы подписчиков без падения приложения.
- Код соответствует стандарту PEP-8 и имеет подробные докстринги.

## Установка и запуск (Локально)

### 1. Клонирование репозитория
```bash
git clone https://github.com/sharapik26/Avar_dictionary.git
cd Avar_dictionary
```

### 2. Установка зависимостей
Проект требует Python версии 3.10 и выше. Установите зависимости:
```bash
pip install -r requirements.txt
```

### 3. Настройка конфигурации
В файле `bot/config.py` укажите ваш `BOT_TOKEN` (получите его у @BotFather в Telegram). 
Для корректной работы Mini App установите `WEBAPP_URL` (должен быть с https).

### 4. Запуск проекта
Проект объединяет API и Бота в один процесс. Запуск осуществляется одной командой:
```bash
uvicorn api.main:app --host 0.0.0.0 --port 8000
```

После запуска сервер будет доступен по адресу `http://localhost:8000`, а Telegram бот начнет принимать сообщения.

## Деплой (Например, на Render.com)
1. Создайте Web Service на Render.com.
2. Подключите ваш GitHub репозиторий.
3. Укажите `Build Command`: `pip install -r requirements.txt`
4. Укажите `Start Command`: `uvicorn api.main:app --host 0.0.0.0 --port $PORT`
5. Добавьте `BOT_TOKEN` в переменные окружения (Environment Variables).
